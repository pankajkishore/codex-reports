Codex Workspace Operating System

The workspace now has a standing rule file, a clear folder structure, GitHub publishing defaults, privacy guardrails, and a report workflow for turning meaningful work into live HTML pages.

Folder Structure

projects/ Real apps, tools, dashboards, and reusable work.
experiments/ Quick prototypes that may later be promoted into projects.
inbox/ Raw incoming files such as screenshots, PDFs, CSVs, and notes.
archive/ Completed or inactive work that should remain available.
templates/ Reusable starter files, prompts, and report patterns.
codex-reports/ The GitHub Pages hub for shareable HTML reports.

Autopilot Rule

When a task creates a meaningful artifact, Codex should organize it, document it, test or build it when applicable, commit it, push it to GitHub when appropriate, and publish or update an HTML report.

GitHub Publishing Defaults

projects/<slug>      -> real repo candidates
experiments/<slug>   -> quick prototypes
codex-reports/       -> public GitHub Pages reports
AGENTS.md            -> standing Codex rules

Privacy Guardrails

Public reports must not include secrets, credentials, private client data, sensitive personal details, raw mailbox contents, or private documents. When a task might expose private information, Codex should create a local draft first and ask before publishing.
